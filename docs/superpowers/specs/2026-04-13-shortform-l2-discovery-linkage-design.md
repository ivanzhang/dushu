# 短篇馆藏 L2 提升与搜索导览联动设计

- 日期：2026-04-13
- 背景：首页、搜索页、分类总览页的入口导览已经补齐，但站内“短篇 / 志怪 / 话本”这条阅读线仍然偏薄。当前 `阅微草堂笔记`、`子不语`、`喻世明言`、`警世通言`、`醒世恒言` 都还是 `L1`，搜索页也还缺少“点一下就能进”的推荐词、作者入口和专题入口。

## 目标

把“短篇轻阅读”这条线一次性做出可见的站感提升：

- 让 5 本短篇馆藏从 `L1` 推到 `L2`
- 让搜索页具备更强的推荐词、作者入口、专题入口联动
- 让专题页和作者入口更自然地承接这批新内容

用户应当能明显感受到：

- 站里不只有长篇镇馆书，也有成体系的短篇入口
- 搜索页不只是一个输入框，而是一个“从词、从作者、从专题进入”的前台
- 短篇馆藏之间已经开始形成互相导流的阅读路径

## 范围

本次范围只覆盖“5 本短篇组 + 搜索导览 + 专题互链”：

- 内容推进：
  - `阅微草堂笔记`
  - `子不语`
  - `喻世明言`
  - `警世通言`
  - `醒世恒言`
- 页面与数据：
  - `src/pages/search.astro`
  - `src/pages/topic/[slug].astro`
  - `src/data/library/topics.ts`
  - `src/data/library/entry-guides.ts` 或新的搜索导览数据文件
  - 相关测试与构建校验
- 抓取与内容：
  - `scripts/fetch-chapter-helpers.mjs`
  - `scripts/fetch-chapters.mjs`
  - 上述 5 本的书籍元数据与章节文件

## 非目标

本轮不做以下事情：

- 不重做 Pagefind 排序算法
- 不做复杂个性化推荐
- 不把这 5 本直接推到 `L3`
- 不扩展到更多新书种
- 不改已有长篇馆藏的整体展示结构

## 设计方案

### 1. 内容波次：先把 5 本短篇组拉到稳定 `L2`

这轮选择“短篇好见效”的一组，是因为它们更适合和搜索、专题、作者入口同时联动。

#### 1.1 目标批次

- `阅微草堂笔记`：补前 `8` 卷
- `子不语`：补前 `8` 卷
- `喻世明言`：补前 `10` 卷
- `警世通言`：补前 `10` 卷
- `醒世恒言`：补前 `10` 卷

总计新增约 `46` 篇可读内容。

#### 1.2 完成度判定

这 5 本完成本轮后统一提升为 `L2`，理由是：

- 已具备一批真实可读篇目
- 目录不再是空壳
- 搜索可以命中书目和正文入口
- 短篇类作品不要求像长篇一样先具备十几章连续主线，也可以用“首批篇目密度”作为 `L2` 的成立依据

### 2. 搜索页增强：从“工具页”升级为“可直接进入的前台”

搜索页本轮除了保留原有导览卡外，再补三组更直接的入口：

- `热门搜词`
- `从作者进入`
- `从专题进入`

#### 2.1 热门搜词

推荐词不只做静态文案，而是做成可以直接触发搜索的入口。

推荐做法：

- 搜索页支持读取 `?q=` 查询参数
- 页面初始化 `PagefindUI` 后，若 URL 中有 `q`，则自动执行该词搜索
- 热门搜词卡片或按钮统一跳到 `/search?q=<关键词>`

这样可以把搜索页变成真正的“词入口页”，并且后续首页、专题页、作者页都能复用这个深链接能力。

推荐的第一批词：

- 题材词：`志怪`、`话本`、`短篇`、`夜读`
- 作品词：`阅微草堂笔记`、`子不语`、`喻世明言`
- 作者词：`冯梦龙`、`纪昀`、`袁枚`

#### 2.2 从作者进入

搜索页新增作者入口卡片，重点承接：

- `冯梦龙`
- `纪昀`
- `袁枚`

每张卡片既能跳作者页，也能附带一个推荐搜索词，用来帮助用户在“逛作者”和“直接搜”之间切换。

#### 2.3 从专题进入

搜索页新增专题入口卡片，至少覆盖：

- `志怪传奇`
- `三言话本`（本轮新增）
- `世情小说`

目标是让“我不知道搜什么”的用户先从主题进入，再回到搜索页用关键词缩小范围。

### 3. 专题互链：给短篇组补一个真正能承接流量的策展入口

当前站内已有 4 个专题，但“三言”这一组还没有独立入口。

本轮新增一个专题：

- 名称：`三言话本`
- slug：`three-yans`

其作用是：

- 把 `喻世明言`、`警世通言`、`醒世恒言` 收成一个完整入口
- 把冯梦龙这个作者入口做实
- 成为搜索页和作者页之间的中间导流层

#### 3.1 专题数据增强

建议给专题数据补两类字段：

- `relatedTopicSlugs`
  - 用于专题之间的互链
- `featuredAuthorSlugs`
  - 用于在专题页展示“适合从哪位作者进入”

这样专题页不只讲“这组书为什么放一起”，还能回答：

- 如果我喜欢这个专题，下一步还可以去哪
- 我可以从哪个作者开始读

#### 3.2 专题页渲染增强

专题页新增两个区块：

- `相关专题`
- `作者入口`

其中：

- `相关专题` 承接“同类主题继续逛”
- `作者入口` 承接“顺着人来读”

这会让专题页和作者页从单向跳转，变成真正的双向联动。

### 4. 抓取与数据层：补齐 5 本短篇书的来源规则

现有抓取脚本已经支持一批长篇和重点馆藏，但尚未覆盖这 5 本。

本轮需要补齐以下规则：

- `阅微草堂笔记`
  - 维基文库子页格式为 `卷1`、`卷2`……
- `子不语`
  - 维基文库子页格式为 `卷1`、`卷2`……
- `喻世明言`
  - 维基文库子页格式为 `第01卷`、`第02卷`……
- `警世通言`
  - 维基文库子页格式为 `第01卷`、`第02卷`……
- `醒世恒言`
  - 维基文库子页格式为 `第01卷`、`第02卷`……

同时要更新这 5 本的元数据：

- `completionLevel: L1 -> L2`
- `featuredTopics`
  - `阅微草堂笔记`、`子不语` 继续挂到 `strange-tales`
  - 三言三部补挂到 `three-yans`

### 5. 数据边界

为了避免把搜索页做成又一坨写死模板，本轮仍采用数据驱动：

- 搜索页新增的热门搜词、作者入口、专题入口应放在数据文件里
- 专题互链与作者入口挂在 `src/data/library/topics.ts`
- 页面模板只负责循环渲染

如果搜索导览数据继续增多，允许把 `src/data/library/entry-guides.ts` 拆出专门的 `search-guides.ts`，但前提是能明显降低文件复杂度；否则优先复用现有入口导览数据文件。

## 数据流

本轮的数据流建议如下：

1. 抓取脚本负责把维基文库子页抓成 Markdown 章节
2. 书籍 JSON 更新真实 `completionLevel` 与 `featuredTopics`
3. 专题数据定义专题书单、关联专题、关联作者
4. 搜索导览数据定义推荐词、作者入口、专题入口
5. 搜索页读取 URL 查询词并触发 `PagefindUI`
6. 专题页与作者页通过数据互链承接新短篇内容

## 测试策略

采用 TDD，并分两层校验：

### 1. 数据与脚本测试

- 在 `tests/library-data.test.ts` 中锁定：
  - 这 5 本的 `completionLevel` 已提升到 `L2`
  - 三言专题存在且书单完整
  - 相关专题 / 作者入口字段存在
- 如脚本辅助函数有新规则，补对应的 helper 测试

### 2. 构建产物校验

在 `tests/build-verify.ts` 中锁定：

- 搜索页出现：
  - `热门搜词`
  - `从作者进入`
  - `从专题进入`
- `dist/topic/three-yans.html` 存在
- 三言专题页出现：
  - `相关专题`
  - `作者入口`

### 3. 全量验证

实现后必须运行：

- `bun run test`
- `bun run build`
- `bun run build:verify`

## 验收标准

- 上述 5 本都具有首批真实篇目，且元数据为 `L2`
- 搜索页支持 `/search?q=关键词` 直达触发搜索
- 搜索页新增 `热门搜词`、`从作者进入`、`从专题进入`
- 新增专题 `三言话本`
- 三言专题页具备 `相关专题` 和 `作者入口`
- 全量测试、构建、构建校验通过
