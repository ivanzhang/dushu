# 短篇三书 L2 厚度均衡设计

- 日期：2026-04-14
- 背景：上一轮已经把短篇线的入口、专题、作者联动补齐，并把 `阅微草堂笔记`、`喻世明言` 提到更高完成度；同时 `子不语`、`警世通言`、`醒世恒言` 也已经具备前 `12` 篇试读与 `L2` 完成度。现在短篇线的问题不再是“有没有入口”，而是这三本支撑书的厚度仍偏薄，书架观感还不够整齐。

## 目标

把 `子不语`、`警世通言`、`醒世恒言` 继续做厚，但保持节奏克制：

- 三本都从前 `12` 篇推进到前 `16` 篇
- 三本的真实完成度继续保持 `L2`
- 不新增模板、不扩入口，只补正文承接
- 让短篇线书架厚度更均衡，便于下一轮再决定是否把某一本单独推进到 `L3`

用户应当能感受到：

- 短篇分馆不是只有两本更厚、三本偏薄，而是整排书开始更整齐
- `子不语` 与“三言”两部支撑书点进去后更像真实可试读书架
- 站点内容推进开始从“先立入口”转向“补厚已有馆藏”

## 方案对比

### 方案 A：只把其中一本推进到 `L3`

优点：

- 更快得到一个单点更强的短篇入口
- 便于在首页或专题页上突出重点

缺点：

- 其余两本会继续显得偏薄
- 短篇整排书的均衡感不足
- 现在就升级到 `L3` 也容易让完成度口径失衡

### 方案 B：三本一起从 `12` 补到 `16`，但继续保持 `L2`（采用）

优点：

- 三本都能同步变厚，书架观感最稳
- 风险小，复用现有抓取规则即可推进
- 保持完成度判定口径一致，不会过早拔高

缺点：

- 这一轮不会立刻出现新的 `L3` 亮点

### 方案 C：继续不动三本，转去做新入口或新书

优点：

- 新鲜感更强
- 可以更快扩馆面

缺点：

- 已有短篇线会继续停在“入口有了、内容略薄”的状态
- 当前站点更需要把已有馆藏做扎实

## 采用方案

采用 **方案 B**：

- `zibuyu`、`jingshitongyan`、`xingshihengyan` 三本一起推进
- 每本都从前 `12` 篇补到前 `16` 篇
- `completionLevel` 继续保持 `L2`
- 本轮不改模板、不扩主题、不新增入口

## 范围

本轮范围：

- `tests/library-data.test.ts`
- `scripts/fetch-chapters.mjs`（只复用，不主动重构）
- `scripts/fetch-chapter-helpers.mjs`（只确认规则复用，不新增配置）
- `src/content/books/zibuyu.json`
- `src/content/books/jingshitongyan.json`
- `src/content/books/xingshihengyan.json`
- `src/content/chapters/zibuyu/`
- `src/content/chapters/jingshitongyan/`
- `src/content/chapters/xingshihengyan/`
- `docs/progress.md`

## 非目标

本轮不做以下事情：

- 不新增专题、分类、搜索入口
- 不修改首页、搜索页、专题页、书籍页模板
- 不把 `子不语`、`警世通言`、`醒世恒言` 提升到 `L3`
- 不调整 `阅微草堂笔记` 与 `喻世明言` 当前完成度
- 不处理版权链路和大规模构建性能验证

## 设计方案

### 1. 用“前 16 篇”作为这一轮的补厚目标

前三轮已经形成了比较稳定的推进节奏：

- 长篇主线常以“前 `12` 章”作为稳定试读门槛
- 短篇组已经先拉到“前 `12` 篇”形成基础试读

这一轮不另起一套标准，而是在已有 `12` 篇基础上继续补到 `16` 篇。这样既能让目录明显变厚，又不会把本轮范围拉得太大。

具体目标：

- `zibuyu`: `12 -> 16`
- `jingshitongyan`: `12 -> 16`
- `xingshihengyan`: `12 -> 16`

### 2. 完成度继续保持 `L2`

本轮的判断重点是“补厚”，不是“升级等级”。

保持 `L2` 的原因：

- 三本虽然会新增 4 篇，但仍属于更稳的试读深度，而不是足以定义为馆内强连续入口的层级
- 与当前 `阅微草堂笔记`、`喻世明言` 已形成的分层关系保持一致
- 方便下一轮再根据用户选择，决定是继续均衡补厚，还是让其中一本单独冲 `L3`

### 3. 只走既有抓取链路，不新加脚本复杂度

这三本的抓取规则已经存在于：

- `scripts/fetch-chapter-helpers.mjs`

因此本轮不新增抓取规则，只做两件事：

1. 先用测试把目标提升到 `16`
2. 再直接复用抓取脚本抓取 `13-16`

预期命令：

- `bun scripts/fetch-chapters.mjs -- zibuyu 13 16`
- `bun scripts/fetch-chapters.mjs -- jingshitongyan 13 16`
- `bun scripts/fetch-chapters.mjs -- xingshihengyan 13 16`

### 4. 页面模板保持不动，依赖真实内容自然接通

本轮不改模板，原因是：

- 书籍页、专题页、分类页已经能基于真实章节目录自动反映馆藏厚度
- 当前缺的不是结构能力，而是真实正文承接数量
- 用最小改动补正文，比继续做模板层变化更符合当前目标

## 测试策略

### 1. 先补数据测试红灯

在 `tests/library-data.test.ts` 先把以下目标写成失败断言：

- `zibuyu >= 16`
- `jingshitongyan >= 16`
- `xingshihengyan >= 16`
- 三本 `completionLevel` 都仍为 `L2`

### 2. 再抓新增正文转绿

使用既有抓取脚本补 `13-16` 篇，确认目录数量增长到目标值。

### 3. 最后跑完整验证

最后统一运行：

- `bun run test`
- `bun run build`
- `bun run build:verify`

## 验收标准

本轮完成后，应满足：

- `src/content/chapters/zibuyu/` 至少有前 `16` 篇
- `src/content/chapters/jingshitongyan/` 至少有前 `16` 篇
- `src/content/chapters/xingshihengyan/` 至少有前 `16` 篇
- 三本 JSON 的 `completionLevel` 仍为 `L2`
- `bun run test` 通过
- `bun run build` 通过
- `bun run build:verify` 通过
- 本轮结果写入 `docs/progress.md`
