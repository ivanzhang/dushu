# 首页、搜索页、分类总览页入口导览增强设计

- 日期：2026-04-13
- 背景：分类页、专题页、作者页已经具备更明确的策展导览能力，但首页、搜索页、分类总览页仍偏“陈列页”或“工具页”，缺少统一的入口语言和新读者导航。

## 目标

把站点最前面的三类入口页补成更像“小型读书馆前台”：

- 首页负责回答“第一次来这里该怎么逛”
- 搜索页负责回答“我不知道搜什么时该怎么进入”
- 分类总览页负责回答“我该先选哪一馆”

## 范围

本次只增强三个入口页，不改现有书籍页、专题页和作者页结构：

- 修改 `src/pages/index.astro`
- 修改 `src/pages/search.astro`
- 修改 `src/pages/category/index.astro`
- 新增或修改导览数据文件
- 修改构建校验
- 追加进度记录到 `docs/progress.md`

## 设计方案

### 1. 首页增强

首页新增三组导览内容：

- `怎么逛这座小馆`
  - 用 3 条最短路径告诉第一次来的读者如何进入
- `按兴趣进入`
  - 用几张卡片把“名著 / 夜读 / 长篇 / 古文入口”区分开
- `馆藏地图`
  - 把分馆、专题、作者三条导览线并列摆出来

目标是让首页除了“看见书”，还能“知道怎么走”。

### 2. 搜索页增强

搜索页保留搜索工具定位，但新增两组内容：

- `如果你不知道搜什么`
  - 给出几个高确定性的快捷入口
- `按读法找书`
  - 用“长篇深读 / 短篇夜读 / 熟书入门 / 非小说入口”这种读法来分流

目标是让搜索页从“工具页”升级成“工具 + 导览页”。

### 3. 分类总览页增强

分类总览页新增两组总览导览：

- `怎么选馆`
  - 告诉读者按兴趣、体量、阅读节奏选哪一馆
- `推荐逛法`
  - 提供几条跨分馆阅读路径

同时让每张分馆卡片多展示一条简短的 `适合谁` 文案。

目标是让分类总览页从“馆列表”升级成“选馆前台”。

## 数据流

采用数据驱动，避免把策展文案直接写死在模板里：

- 新增一个导览数据文件，集中管理首页、搜索页、分类总览页的入口文案
- 分类总览页复用现有分类元数据里的 `audience`
- 页面模板只负责循环渲染，不负责硬编码长文案

## 测试策略

采用 TDD：

1. 先在 `tests/build-verify.ts` 中补失败校验，锁定三个入口页必须出现的新导览区块
2. 如新增导览数据文件，再补对应的数据测试
3. 实现后运行：
   - `bun run test`
   - `bun run build`
   - `bun run build:verify`

## 验收标准

- 首页至少出现：`怎么逛这座小馆`、`按兴趣进入`、`馆藏地图`
- 搜索页至少出现：`如果你不知道搜什么`、`按读法找书`
- 分类总览页至少出现：`怎么选馆`、`推荐逛法`
- 分类总览页的分馆卡片中出现 `适合谁`
- 全量测试、构建、构建验证通过
