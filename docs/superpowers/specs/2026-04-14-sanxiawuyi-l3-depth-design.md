# 三侠五义 L3 深读入口推进设计

- 日期：2026-04-14
- 背景：今天早些时候已经把 `三侠五义`、`说岳全传`、`隋唐演义` 三本统一补到前 `12` 回，并把三本真实完成度提升到 `L2`。这一步解决了公案 / 英雄线“有入口但没正文承接”的问题。下一步不宜再三本一起平均铺开，而应先挑一本最适合做强单点入口的作品，率先推进到 `L3`。

## 目标

先把 `三侠五义` 做成公案侠义线的第一个强深读入口：

- 把 `sanxiawuyi` 从前 `12` 回推进到前 `20` 回
- 把 `src/content/books/sanxiawuyi.json` 的 `completionLevel` 从 `L2` 提升到 `L3`
- 保持 `说岳全传`、`隋唐演义` 现状不动，继续停在前 `12` 回、`L2`
- 不改模板，只通过真实章节与元数据变化让页面自然升级

用户应当能感受到：

- `公案侠义` 线终于有了一本真正更能连续往下读的主打作品
- `三侠五义` 不再只是“有目录”，而是开始具备更稳定的深读体验
- 英雄线和公案线虽然都已有承接，但会优先把最适合做门面的那一本先做强

## 方案对比

### 方案 A：先把 `三侠五义` 推到前 `20` 回并升到 `L3`（采用）

优点：

- `三侠五义` 是 `公案侠义` 线最直接的门面书，收益最集中
- 包公公案 + 江湖侠义的题材辨识度高，适合先做强单点入口
- 改动集中，最适合作为本轮继续推进的下一步

缺点：

- `英雄传奇` 线会继续停留在均衡但不够深的状态

### 方案 B：先推 `说岳全传`

优点：

- 岳飞题材识别度高，忠义叙事更容易承接一部分读者
- 能进一步补强英雄线

缺点：

- 同时挂在 `historical-epics` 与 `heroic-tales`，入口角色更分散
- 作为“下一本先做强”的优先级不如 `三侠五义` 集中

### 方案 C：先推 `隋唐演义`

优点：

- 篇幅宏阔，人物群像和历史风云感更强
- 对历史演义线也有帮助

缺点：

- 更像历史演义线的铺面书，而不是公案 / 英雄线最尖锐的入口
- 题材焦点相对不如 `三侠五义` 明确

## 采用方案

采用 **方案 A**：

- 本轮只推进 `三侠五义`
- 目标从前 `12` 回补到前 `20` 回
- `completionLevel: L2 -> L3`
- `说岳全传`、`隋唐演义` 暂不变化

## 范围

本轮范围：

- `tests/library-data.test.ts`
- `scripts/fetch-chapters.mjs`（只复用，不主动重构）
- `src/content/books/sanxiawuyi.json`
- `src/content/chapters/sanxiawuyi/`
- `docs/progress.md`

## 非目标

本轮不做以下事情：

- 不继续推进 `说岳全传`、`隋唐演义`
- 不新增专题、搜索入口或分类导览文案
- 不修改首页、搜索页、专题页、书籍页模板
- 不重构抓取脚本
- 不处理版权链路和大规模构建性能验证

## 设计方案

### 1. 用“前 20 回”作为 `三侠五义` 冲 `L3` 的最小门槛

前几轮推进里，站内已经形成两层比较稳定的门槛：

- 前 `12` 回 / 篇：稳定试读入口，对应 `L2`
- 比 `12` 更厚一层的连续阅读：适合作为 `L3` 的候选

本轮不把 `三侠五义` 一口气推得过深，而是用前 `20` 回作为最小但清晰的升级门槛。这样既能明显区别于当前的 `L2` 状态，也能控制范围，让这轮仍然是一个独立可交付的小阶段。

具体目标：

- `sanxiawuyi`: `12 -> 20`

### 2. 只把 `三侠五义` 升到 `L3`

这轮的核心是“先立一个更强的公案侠义深读入口”。

因此：

- `sanxiawuyi`: `L2 -> L3`
- `shuoyuequanzhuan`: 保持 `L2`
- `suitangyanyi`: 保持 `L2`

这样做的好处是：

- 分层更明确：不是三本平均变厚，而是先挑一本门面书做强
- 后续路径也清晰：下一轮可以再决定英雄线由哪一本接着冲 `L3`

### 3. 继续走既有抓取链路，只补 `13-20` 回

`三侠五义` 的抓取规则已存在，本轮无需再补规则，只需复用抓取脚本补新增回目。

预期命令：

- `bun scripts/fetch-chapters.mjs -- sanxiawuyi 13 20`

生成结果应为：

- `src/content/chapters/sanxiawuyi/013.md` 到 `020.md`

### 4. 页面模板保持不动，依赖真实内容自然接通

本轮不改模板，原因是：

- 页面层已经能根据真实章节数量和书籍完成度显示状态
- 当前缺的不是结构，而是更厚的真实阅读承接
- 只要章节和 `completionLevel` 到位，公案线入口自然就会更像样

## 测试策略

### 1. 先补数据测试红灯

在 `tests/library-data.test.ts` 先补或调整以下断言：

- `sanxiawuyi >= 20`
- `shuoyuequanzhuan >= 12`
- `suitangyanyi >= 12`
- `sanxiawuyi.completionLevel === 'L3'`
- 其余两本仍为 `L2`

### 2. 再抓正文和更新元数据转绿

补齐 `13-20` 回并把 `sanxiawuyi` 的完成度升到 `L3`。

### 3. 最后跑完整验证

最后统一运行：

- `bun run test`
- `bun run build`
- `bun run build:verify`
