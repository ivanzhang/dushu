# 分类页、专题页、作者页策展增强设计

- 日期：2026-04-13
- 背景：当前站点已经完成 20 本首轮馆藏与基础聚合页，但分类页、专题页、作者页仍以“书单页”居多，缺少明确的策展说明、阅读入口和馆内导览关系。

## 目标

把现有三类聚合页从“能列书”提升到“像小型读书馆的馆员导览页”：

- 分类页不只展示馆藏，还要说明“这一馆适合谁、怎么逛、和隔壁馆有什么区别”
- 专题页不只展示主题书单，还要说明“为什么做这个专题、该从哪本开始”
- 作者页不只展示作者作品，还要说明“这位作者在馆里的位置、气质、关联去向”

## 范围

本次只做当前首轮馆藏范围内的内容增强，不扩新页面类型：

- 修改 `src/data/library/categories.ts`
- 修改 `src/data/library/topics.ts`
- 修改 `src/data/library/authors.ts`
- 修改 `src/pages/category/[slug].astro`
- 修改 `src/pages/topic/[slug].astro`
- 修改 `src/pages/author/[slug].astro`
- 补齐测试与构建校验
- 追加进度记录到 `docs/progress.md`

## 设计方案

### 1. 分类页增强

分类元数据新增三类信息：

- `audience`：适合什么读者进入这一分馆
- `entryPath`：建议的入馆顺序，使用短句数组
- `crossShelfLinks`：可顺手转去的相邻分馆及理由

分类页新增三个内容块：

- `适合谁读`
- `入馆路线`
- `邻馆串逛`

这样分类页会从“图书分类页”变成“分馆导览页”。

### 2. 专题页增强

专题元数据新增三类信息：

- `curatorNote`：专题缘起，说明为什么要把这些书放在一起
- `readingPathBookSlugs`：专题内的建议阅读路线
- `relatedCategorySlugs`：与本专题关联的分馆入口

专题页新增三个内容块：

- `策展缘起`
- `阅读路线`
- `相关分馆`

这样专题页不会只剩一组卡片，而会更像馆内策展展签。

### 3. 作者页增强

作者元数据扩展为首轮馆藏全覆盖，并新增：

- `styleTags`：作者气质标签
- `libraryRole`：这位作者在馆内承担的角色
- `entryBookSlug`：第一次阅读优先进入的书
- `relatedCategorySlugs`：建议顺手跳转的分馆
- `relatedTopicSlugs`：建议顺手跳转的专题

作者页新增四个内容块：

- `馆内定位`
- `气质标签`
- `从哪本开始`
- `顺手再逛`

这样作者页会从“作者简介页”变成“作者展架页”。

## 数据流

- 页面继续从 `astro:content` 读取真实书目
- 策展文本继续从 `src/data/library/*.ts` 读取
- 页面渲染时通过 `src/lib/library.ts` 里的元数据查询函数进行组装
- 不改变现有书卡、排序、完成度与馆藏等级逻辑

## 测试策略

采用 TDD：

1. 先在 `tests/library-data.test.ts` 中补失败测试，锁定新增元数据字段与作者资料覆盖率
2. 再在 `tests/build-verify.ts` 中补失败校验，锁定新页面区块必须出现在构建产物中
3. 通过后运行：
   - `bun run test`
   - `bun run build`
   - `bun run build:verify`

## 风险与约束

- 当前工作区不是干净状态，只修改本次明确涉及的页面、数据和测试文件
- 不触碰用户已有部署链路与其他未提交改动
- 不做新的页面体系重构，避免把本次“内容增强”扩散成大规模 UI 重写

## 验收标准

- 分类页至少出现：`适合谁读`、`入馆路线`、`邻馆串逛`
- 专题页至少出现：`策展缘起`、`阅读路线`、`相关分馆`
- 作者页至少出现：`馆内定位`、`气质标签`、`从哪本开始`、`顺手再逛`
- 作者策展资料覆盖当前首轮馆藏涉及的全部作者 slug
- 全量测试、构建、构建验证通过
